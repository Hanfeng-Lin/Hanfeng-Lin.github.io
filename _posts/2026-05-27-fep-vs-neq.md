非平衡自由能模拟（Non-equilibrium Free Energy Simulation, NEQ）是一类用“非平衡过程中的做功”来估计平衡态自由能差的方法。它在蛋白-配体相对结合自由能（RBFE）计算中，常被拿来和传统 FEP/TI/BAR 工作流比较，因为两者都在回答同一个问题：把配体 A 变成配体 B，结合自由能会改变多少？

如果你已经会做 FEP，第一次听到 NEQ 时，大概率会有一点困惑：自由能不是平衡态性质吗？为什么一个故意把体系拉离平衡、只跑几十皮秒的过程，最后反而还能算出平衡态的 $\Delta G$？

这个疑问很正常，因为 FEP 训练出来的直觉是：要在一串 $\lambda$ 窗口里慢慢走，每个窗口都要尽量采到平衡，窗口之间还要有足够的 phase space overlap。突然有人说“不用那么多中间窗口，先把两个端态采好，然后从 A 快速拉到 B、再从 B 快速拉回 A，统计这些非平衡轨迹的做功就行”，听起来像是在破坏 FEP 最珍贵的假设。

NEQ 的关键并不是无视平衡态，而是换了一种抵达平衡态自由能的路径。FEP 问的是：在同一个构象上，把哈密顿量从 $U_A$ 瞬间换成 $U_B$，能量差是多少？NEQ 问的是：如果我真的用一段有限时间把体系从 A 驱动到 B，这一路外界对体系做了多少功？Jarzynski 等式和 Crooks 涨落定理告诉我们，只要起点来自正确的平衡分布，这些非平衡做功里仍然包含着平衡态 $\Delta G$。

所以这篇文章从 FEP 使用者熟悉的概念出发，把几个最容易卡住的点讲清楚：为什么 FEP 可以说是“瞬时跳跃”，NEQ 里的“短拉伸”到底拉的是什么，Crooks/BAR 为什么能从正反向做功分布里抽出自由能，以及在算力和准确性上，NEQ 到底什么时候值得替代或补充 FEP。

## 一句话版本

如果只记住一句话，可以这样理解：

**FEP 把 A 到 B 的变化切成很多个平衡态窗口，在相邻窗口之间做“冻结坐标”的瞬时能量差；NEQ 只充分采样 A 和 B 两个端态，然后从端态构象出发，在有限时间内连续改变哈密顿量，并记录这个非平衡过程的做功。**

也就是说，FEP 更像是在空间上切片，NEQ 更像是在时间上积分。

| 维度 | FEP / BAR | NEQ / Crooks-BAR |
|---|---|---|
| 核心对象 | 相邻 λ 窗口的势能差 $\Delta U$ | A 到 B、B 到 A 的非平衡做功 $W$ |
| 采样重点 | 多个中间 λ 窗口的平衡采样 | 两个物理端态的充分采样 |
| 计算结构 | 多窗口、常配合 HREX | 大量短轨迹，彼此独立 |
| 主要风险 | 中间窗口采样不充分、相空间不重叠 | 耗散功过大、正反向做功分布不重叠 |
| 更适合 | 标准小分子 R 基团微调、产业化 SAR | 端态构象清楚、需要高并发、局部构象重排明显的体系 |

## FEP 为什么被说成“瞬时跳跃”

很多人第一次听到“FEP 是瞬时跳跃”会觉得奇怪：标准 FEP 明明设置了很多个 λ 窗口，比如 $\lambda=0,0.05,0.10,\dots,1.0$，这不就是渐进变化吗？

这里要区分**宏观工作流**和**最小统计单元**。

在宏观流程上，FEP 的确是渐进的。我们人为插入许多中间态，让相邻窗口足够相似，并在每个窗口里做平衡态 MD 采样。但在相邻窗口之间计算自由能差时，FEP 使用的是 Zwanzig 方程：

$$
e^{-\beta \Delta G_{i\to j}}
=
\left\langle e^{-\beta \left[U_j(r)-U_i(r)\right]} \right\rangle_i
$$

这里的平均 $\langle\cdot\rangle_i$ 表示构象 $r$ 来自状态 $i$ 的平衡系综。关键点在于：对某一个具体构象 $r$，我们计算 $U_j(r)-U_i(r)$ 时，坐标是不动的。系统没有时间弛豫，也没有沿着新势能面跑一小段动力学。我们只是把同一个原子坐标，放到另一个哈密顿量下面重新打分。

这就是“瞬时跳跃”的含义。它不是说整个 FEP 流程粗暴地从 $\lambda=0$ 跳到 $\lambda=1$，而是说每一次相邻窗口的微扰估计，本质上都是一个冻结坐标下的垂直跃迁。

这也解释了 FEP 为什么如此依赖相空间重叠。如果状态 $i$ 里常见的构象，在状态 $j$ 里都是高能构象，那么 $U_j-U_i$ 的指数平均会被极少数低功事件支配，收敛会非常差。所以生产级 FEP 往往需要：

- 足够密的 λ 窗口；
- soft-core potentials 来避免原子生成/湮灭时的奇异性；
- HREX 或 REST 之类的增强采样；
- 闭环分析来识别热力学网络中的不一致。

## NEQ 的理论基础：把能量差换成做功

NEQ 的出发点不是“在两个相邻平衡态之间瞬时重打分”，而是“让系统真的经历一个含时变化过程”。

假设我们从状态 A 的一个平衡构象出发，在 50 ps 内把 $\lambda$ 从 0 连续推到 1。模拟过程中，势能函数在变，原子坐标也在随时间演化。这个过程通常远离平衡，因此每条轨迹都会产生一个非平衡做功 $W$。

Jarzynski 等式告诉我们，即使这个过程是非平衡的，只要初始构象来自状态 A 的平衡分布，很多条 A 到 B 的轨迹的指数平均仍然给出平衡态自由能差：

$$
e^{-\beta \Delta G}
=
\left\langle e^{-\beta W_{A\to B}} \right\rangle_A
$$

这条公式非常漂亮，但直接用它并不好算。因为指数平均会被罕见的低功轨迹支配。如果拉伸很快，耗散功很大，那些 $W<\Delta G$ 的轨迹极少出现，单向 Jarzynski 估计会非常不稳定。

Crooks 涨落定理进一步利用了正向和反向两组轨迹：

$$
\frac{P_F(W)}{P_R(-W)}
=
e^{\beta(W-\Delta G)}
$$

其中 $P_F(W)$ 是 A 到 B 的做功分布，$P_R(-W)$ 是 B 到 A 做功分布取镜像后的分布。一个直观结论是：当 $W=\Delta G$ 时，等式右边为 1，所以正向做功分布和反向镜像分布的交点就是自由能差。

实际计算里通常不会真的只用肉眼找交点，而是使用 Bennett Acceptance Ratio (BAR) 或最大似然估计，把两组做功分布合并起来估计 $\Delta G$。这也是为什么 NEQ 经常被描述为 Jarzynski-Crooks-BAR 路线。

## “短拉伸”到底在拉什么

NEQ 里的“拉伸”容易让人误会。它不是 Steered MD 里把配体从口袋里物理拉出来，也不是给某两个原子加弹簧。

这里拉的是**化学空间里的 λ 旋钮**。

比如我们要比较两个配体：

- A：苯环上是 `-H`
- B：苯环上是 `-CH3`

在 NEQ 短拉伸中，模拟从 A 的平衡构象开始，接下来 50 到 200 ps 内，力场参数被连续改变：原来对应 `-H` 的相互作用逐渐消失，`-CH3` 对应的范德华半径、电荷、键角、二面角等相互作用逐渐出现。与此同时，周围的水、蛋白侧链、配体自身都按照牛顿方程继续运动。

如果用一个粗糙但有用的比喻：FEP 像是慢慢给气球打气，每打一小口气就等房间里的人重新站稳；NEQ 像是在很短时间内快速把气球打大，周围的人来不及优雅让位，于是系统产生碰撞、弛豫和耗散功。NEQ 不试图避免这些非平衡效应，而是把它们记录成做功分布，再用 Crooks/BAR 从统计上把平衡态自由能提取出来。

短拉伸不能太短，也不能太长：

- 如果太短，例如 1 ps，体系几乎没有时间响应，耗散功巨大，正反向分布可能完全分离。
- 如果太长，例如 10 ns，过程接近准静态，统计噪音会下降，但每条轨迹太贵，NEQ 的高并发优势就被吃掉了。
- 经验上，很多蛋白-配体 RBFE 场景会尝试 50 到 200 ps 的非平衡转变，然后用正反向做功分布是否重叠来判断是否可用。

## 一个标准 NEQ RBFE 工作流

以蛋白-配体相对结合自由能为例，NEQ 的完整计算仍然需要水相和复合物相两套炼金路径。最终目标是：

$$
\Delta\Delta G
=
\Delta G_\text{complex}
-
\Delta G_\text{water}
$$

一个常见工作流如下。

**1. 端态平衡采样**

分别对配体 A 和配体 B 做端态 MD，且水相和蛋白-配体复合物相都要做。NEQ 能不能稳定，很大程度上取决于这一步有没有把端态的重要构象采出来。

**2. 从端态轨迹抽帧**

从 A 的平衡轨迹中抽取几十到几百个构象，作为 A 到 B 的初始点；从 B 的平衡轨迹中抽取同样数量的构象，作为 B 到 A 的初始点。

**3. 批量非平衡转变**

对每个初始构象启动一条短轨迹，在 50 到 200 ps 内把 $\lambda$ 从 0 推到 1，或从 1 推回 0。每条轨迹记录总做功 $W$。这些短任务之间几乎没有通信，可以非常自然地分发到大量 GPU 或集群节点上。

**4. 做功分布分析**

分别得到水相和复合物相的正向/反向做功分布，用 BAR 或最大似然估计得到 $\Delta G_\text{water}$ 和 $\Delta G_\text{complex}$，最后相减得到 $\Delta\Delta G$。

**5. 质量控制**

NEQ 的一个优点是 QC 很直观：画出正向做功分布和反向镜像分布。如果两者几乎没有重叠，结果就不应该被相信。此时要么延长拉伸时间，要么增加端态采样，要么引入中间态，把一个太大的变化拆成几段。

## FEP 和 NEQ 的算力差异

FEP 当然可以并行。生产环境里，16 个 λ 窗口分给 16 张 GPU 同时跑是非常常见的做法。如果再加 HREX，不同窗口之间还会周期性交换构象。

所以“FEP 是串行的”这句话并不准确。更准确的说法是：**FEP 的并行是强耦合并行，NEQ 的并行是松耦合并行。**

FEP 的窗口之间虽然可以同时跑，但它们往往需要：

- 相似的运行速度；
- 稳定的低延迟通信；
- 同步推进的 replica exchange；
- 一整套窗口都收敛后才能给出可靠网络结果。

NEQ 的短轨迹则更像一堆独立小任务。某条 A 到 B 轨迹在一张 GPU 上跑完，另一条 B 到 A 轨迹在另一台机器上跑完，中间不需要相互等待。只要最后能收集所有 work values，就可以进入分析。

这带来一个很实际的差别：如果你有一批碎片化 GPU 资源，NEQ 更容易把它们吃满；如果你有一套稳定、通信快、适合 HREX 的 HPC 环境，成熟 FEP 流程仍然非常强。

## 准确性：两者失败的方式不同

FEP 和 NEQ 都不是魔法。它们的误差来源不同。

FEP 最怕的是中间窗口没有真正平衡。比如配体突变导致蛋白 loop 或关键侧链需要重排，但某些 λ 窗口卡在局部极小值里，有限模拟时间内没有跨过能垒。这会让相邻窗口的 $\Delta G$ 拼接出现系统误差。

NEQ 最怕的是耗散功太大。如果 A 和 B 差异过大，短拉伸会制造严重原子碰撞或强烈电荷重排，正反向做功分布完全分开，BAR 没有足够重叠区域可用。此时即使轨迹数很多，也只是更精确地测量了一个不可用的非平衡过程。

一个实用决策表可以这样写：

| 场景 | 更推荐 | 原因 |
|---|---|---|
| 同一母核的小 R 基团微调 | FEP / FEP+ | 工程成熟，力场和网络分析生态好 |
| 分子胶或 PROTAC 三元复合物，界面有局部柔性 | NEQ 可重点尝试 | 端态构象采样后，短轨迹更容易高通量探索 |
| 电荷变化很大、原子生成/湮灭剧烈 | FEP 更稳 | 多窗口和 soft-core 可以更平滑地稀释高能过程 |
| 需要快速扫大量候选突变，且 GPU 资源碎片化 | NEQ | 大量短任务天然适合高并发调度 |
| 两个配体骨架差异很大 | 都要谨慎 | 可能需要拆路径、中间态或重新定义 mapping |

## FEP 和 FEP+ 不是一回事

FEP 是方法，FEP+ 是产品。

更具体地说，FEP+ 是 Schrödinger 围绕 FEP/BAR/HREX/REST 等技术做出的商业化完整管线。它真正强的地方不只是“会算 FEP”，而是把自由能计算里最容易出错、最耗专家时间的环节工程化了。

| 维度 | 自建 FEP 流程 | Schrödinger FEP+ |
|---|---|---|
| 力场 | GAFF、CGenFF、OpenFF 等，需要用户判断参数质量 | OPLS 系列深度集成，参数体系高度工程化 |
| 拓扑映射 | 依赖 pmx、LOMAP、手工检查和大量调参 | FEP Mapper 自动规划微扰网络 |
| 采样 | 常规 MD、HREX，增强采样需自行设计 | HREX + REST 等能力集成度高 |
| 误差诊断 | 需要自己做闭环、重加权和网络检查 | 自动 cycle closure 和网络级误差分析 |
| 适用人群 | 有计算化学经验、愿意维护管线的人 | 需要稳定 SAR 预测的工业研发团队 |

所以讨论“FEP vs FEP+”时，本质上是在比较“统计力学方法”与“围绕该方法构建的工业级软件系统”。这就像比较发动机原理和一台已经能上路的车：前者解释为什么能动，后者解决怎么稳定、快速、少踩坑地到达目的地。

## NEQ 目前有哪些平台选择

NEQ 生态还没有出现一个像 FEP+ 那样占据绝对主导地位的一键式商业黑盒。更常见的是学术开源工具和自建工作流。

**GROMACS + pmx**

这是目前最经典、最成熟的 NEQ RBFE 路线之一。pmx 可以生成 hybrid topology，配合 GROMACS 做端态采样和非平衡转变，并提供 work value 分析、BAR 估计和分布图。对于已经熟悉 GROMACS 的人，这是最值得优先尝试的路线。

**OpenMM + perses / 自定义 NCMC 工作流**

OpenMM 的优势是 Python API 和可编程性。perses 以及 Chodera lab 生态里的工具更偏向可组合的相对自由能与 NCMC 框架。如果目标是把自由能计算接到自己的后端服务、虚拟筛选平台或自动化调度系统里，OpenMM 路线会更灵活。

**BioExcel Building Blocks (BioBB)**

BioBB 更像工作流胶水。NEQ 的难点不只是单条轨迹怎么跑，而是几百条短轨迹如何生成、调度、收集、分析。BioBB 适合把 GROMACS、pmx、Snakemake 或其他工作流管理器串起来，减少临时文件和脚本管理的混乱。

**OpenEye / NCMC 相关商业方案**

OpenEye 不是典型的“纯 NEQ RBFE”平台，但 Nonequilibrium Candidate Monte Carlo (NCMC) 和相关采样方法在处理水网络、口袋重排、局部构象跳跃时很有价值。如果预算和许可证允许，可以把它看作商业生态里与非平衡采样思想相邻的一类方案。

## 最后的判断标准

我会把 FEP 和 NEQ 看作互补工具，而不是谁替代谁。

FEP 的优势是成熟、稳健、生态完整，尤其是 FEP+ 这样的工业化管线已经把很多工程细节打磨得很好。它适合标准 SAR、小步突变和需要可审计闭环网络的项目。

NEQ 的优势是并行粒度细、质量控制直观，并且在端态构象明确但中间态容易卡住的体系里很有潜力。它适合高通量、端态驱动、需要快速试探一批局部化学变化的场景。

真正实用的策略不是先站队，而是先问三个问题：

1. A 和 B 的化学变化是否足够小，mapping 是否可靠？
2. 端态构象是否已经采样充分，尤其是蛋白 loop、水网络和关键侧链？
3. 正反向 work distributions 是否有足够重叠？

如果这三个问题回答得清楚，FEP 和 NEQ 都可以成为可靠的自由能工具。如果回答不清楚，换一个更高级的软件名字也救不了结果。
